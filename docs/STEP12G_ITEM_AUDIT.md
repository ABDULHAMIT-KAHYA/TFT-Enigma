# Executive Summary

The raw CommunityDragon cache contains 3599 item records. Most records are augments or macro/content objects, not direct combat items.

The runtime item model supports passive stats and triggered `AbilityEffect` lists, and `ItemSystem` can execute `OnCombatStart`, `OnAttack`, `OnHit`, `OnCrit`, `OnCast`, `OnLowHealth`, and `OnDeath` item triggers. Current normalized item data does not use this triggered runtime path: checked-in `data/items/*.json` has 2962 normalized item files, all legacy-shaped without `itemCategory` metadata and with no triggered effects.

The raw cache contains many mechanics that could be mapped later, but most require gates or targeting not yet represented in item import:

- cooldowns
- once-per-combat gates
- auras
- shields
- healing
- damage procs
- execute effects
- summon/spawn/clone effects
- macro-only rewards
- consumable/anvil/loot behavior

# Methodology

Raw category counts use the same generic source-text ordering as the importer:

1. `Augment`: `augment`, `/augments/`, `hexcore`
2. `Anvil`: `anvil`
3. `RadiantItem`: `radiant`
4. `Artifact`: `artifact`, `/artifacts/`
5. `SupportItem`: `support`
6. `Emblem`: `emblem`
7. `Consumable`: `consumable`, `consumables`, `gold`, `reroll`, `xp`, `shop`
8. `LootObject`: `loot`, `orb`, `chest`
9. `CombatItem`: build path, composition, or non-empty effect variables
10. `Unknown`

Effect classification is heuristic over raw effect variables plus source descriptions/tooltips. A record can appear in more than one effect class.

# Runtime Support Today

`Item` currently contains:

- `passiveStats`
- `triggeredEffects`
- shared `ContentMetadata`

`ItemSystem` currently applies passive stats and can execute triggered `AbilityEffect`s for:

- `OnCombatStart`
- `OnAttack`
- `OnHit`
- `OnCrit`
- `OnCast`
- `OnLowHealth`
- `OnDeath`

Triggered execution supports:

- simple damage formulas
- max-HP percent damage field
- healing from damage on hit
- status application with max stacks
- simple area target resolution
- physical crit handling for item damage
- self/single-target/area targeting through existing `AbilityEffect` fields

Importer limitation:

- `writeItemNormalizedFromCdragon` writes `triggeredEffects: []` for all items.
- It maps only stat-looking raw variables into `passiveStats`.
- It preserves raw variables and import warnings when generated by the current importer path, but checked-in normalized items were not regenerated in this audit.

# Highest-Impact Findings

1. Raw item data is dominated by augments: 2415 of 3599 records.
2. Only 445 raw records classify as direct `CombatItem` using current importer rules.
3. 2377 raw records have non-empty effect variables that could at least preserve passive/stat or macro payloads.
4. 2464 raw records are unsupported by current item normalization because they are trigger, macro, targeting, selection, or object records.
5. Raw text indicates substantial triggered mechanics:
   - 116 `OnHit`
   - 28 `OnAttack`
   - 105 `OnCast`
   - 167 `OnCombatStart`
   - 484 cooldown-like effects
   - 43 once-per-combat effects
6. Raw item mechanics also include 144 aura-like records, 117 shield records, 666 heal records, 676 damage records, 48 execute records, and 133 summon/spawn/clone records.
7. Current checked-in normalized items do not preserve `itemCategory`, because items were not regenerated after the Step 12D classifier.
8. Current checked-in normalized items have no `triggeredEffects`, so runtime item trigger execution exists but is unused by live item data.

# Safe Future Mapping Buckets

Safe later, after validation fixtures:

- Passive stat variables already supported by `StatusEffect`.
- Simple `OnCombatStart` self/allied stat effects.
- Simple `OnHit` or `OnAttack` damage using existing `AbilityEffect`.
- Simple `OnCast` self buffs or target damage.
- Simple shields and heals where targeting is self/current target/allies and no cooldown gate is needed.

Blocked until more vocabulary exists:

- cooldown gates
- once-per-combat gates
- deterministic auras
- item-specific target selection
- item stacking/replacement rules
- macro economy rewards
- anvils/loot/consumable behavior
- summons/clones/spawns
- random item/champion generation
- item transformations/reforging/removal

# Risks

- Effect classification is heuristic from raw variables and natural-language descriptions; it is suitable for planning, not executable mapping.
- Raw cache includes many historical/set-specific augment records, so item counts are broader than current live combat items.
- Several records contain opaque hash variable names; those need metadata preservation and fixture tests before mapping.
- Normalized item files are not regenerated, so raw projected category counts and current runtime-loaded item counts intentionally differ.
